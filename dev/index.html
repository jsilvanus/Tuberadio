How it works

      

        Ingest: A fan-out project pushes the YouTube stream to
        rtmp://your-server:1935/live/stream.
        Nginx RTMP receives it and calls on-publish.sh.
      

      

        Processing (FFmpeg): FFmpeg strips the video track and produces two outputs
        simultaneously — a rolling HLS audio playlist at /hls/stream.m3u8 (4-second
        segments, 10-segment window) and a timestamped full-session MP3 archive under
        /archive/.
      

      

        Serving: Nginx HTTP (port 8080) serves the HLS segments and archives with
        CORS headers.  A tiny Node.js process on port 3000 provides the
        /api/status and /api/archive JSON endpoints the widget polls.
      

      

        Widget: tuberadio.js is a self-contained vanilla JS file with
        zero production dependencies.  It auto-initialises any element with
        data-tuberadio, polls for stream status, streams via HLS (native or HLS.js),
        and shows a downloadable archive list.
